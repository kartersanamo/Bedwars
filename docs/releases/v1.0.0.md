## Bedwars v1.0.0

Initial MVP release of the Bedwars plugin, providing a complete playable game loop on a single Spigot/Paper instance.

### Highlights

- **Full BedWars gameplay**
  - Join arenas from a lobby using `/bedwars join`.
  - Automatic lobby waiting and countdown when minimum players are present.
  - Teleport to team islands with starting gear when the game begins.
  - Resource generators for iron, gold, diamonds, and emeralds.
  - Bed breaking, respawn logic, and final kills.
  - Automatic win detection and simple victory sequence.
  - Arena reset between games using an efficient snapshot/restore system.

- **World & arena management**
  - Void world with prebuilt islands used as a template.
  - Per‑arena configuration files under `plugins/Bedwars/arenas/` (`world`, spawns, beds, generators, region).
  - Snapshot-based map restore: only blocks changed during a game are reset, not the entire world folder.

- **Teams and balancing**
  - Configurable teams with colors, spawns, beds, and shop NPCs.
  - Simple team balancer assigns players to the smallest team first.

- **Economy and generators**
  - Iron & gold generators on team islands.
  - Diamond generators on side islands.
  - Emerald generators at center.
  - Global timings and per‑type caps configured via `generators.yml`.

- **Shop (MVP)**
  - Basic in‑game item shop opened via villager interaction.
  - Minimal example items (e.g., wool blocks) purchasable with iron.
  - Designed for easy extension with more categories and items.

- **Sidebar and UI**
  - Per‑player scoreboard showing:
    - Arena display name
    - Current player count
    - Team bed status (alive / destroyed)
  - Uses the internal `SidebarService` and scoreboard API.

- **SQLite statistics**
  - Embedded SQLite DB (`bedwars.db`) with a `player_stats` table.
  - Tracks:
    - Games played
    - Wins
    - Kills and deaths
    - (Schema is ready for final kills and beds broken; basic wiring is in place.)
  - Stats cached in memory and flushed to disk at game end and on plugin disable.

### Configuration

- **Global config**: `config.yml`
  - Lobby world and optional explicit spawn.
  - Default min/max players, team size.
  - Respawn delay, void Y level.
  - Database file name and debug options.

- **Generators**: `generators.yml`
  - Tick intervals and item caps per generator type (iron, gold, diamond, emerald).

- **Sounds**: `sounds.yml`
  - Logical, configurable sounds for key events (bed break, kills, countdown, start, victory, defeat, generator pickup).

- **Arenas**: `arenas/<arenaId>.yml`
  - World name, display name, enabled flag.
  - Min/max players and team size per arena.
  - Lobby and spectator spawns.
  - Team definitions (color, spawn, bed, iron/gold generators, shop NPC).
  - Diamond and emerald generator locations.
  - Arena region bounding box for snapshot/restore.

### Commands

- **`/bedwars join [arena]`**
  - Join a specific arena or the best available one.
  - Permission: `bedwars.join`.

- **`/bedwars leave`**
  - Leave the current arena and return to the lobby spawn.
  - Permission: `bedwars.leave`.

- **`/bedwars start`**
  - Force start the arena you are in (skips the countdown).
  - Permission: `bedwars.admin.start`.

Future releases may add more admin and utility commands, including reload and setup-wizard flows.

### Known Limitations in v1.0.0

- No in‑game arena setup wizard yet; arenas must be configured via YAML.
- Shop is intentionally minimal and does not yet match the full Hypixel/BedWars1058 feature set.
- Sidebar shows core game information but not per‑player detailed stats.
- Rejoin behavior is rudimentary; more robust disconnect/rejoin handling is planned for later versions.
- No Bungee/proxy support; this version targets single Spigot/Paper instances only.

### Upgrade and compatibility

- Built for Spigot/Paper 1.21 with Java 21.
- Uses an embedded SQLite database; no external DB setup is required.
- Future versions will aim to keep configuration formats backward compatible where possible.
